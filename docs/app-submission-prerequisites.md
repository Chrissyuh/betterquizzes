# BetterQuizzes app submission prerequisites

> This checklist is based on the current project architecture and general MCP/app-hosting requirements. Verify against the latest ChatGPT Apps/OpenAI submission docs before final submission.

## 1. Host the MCP server on a stable public HTTPS URL

For development, ngrok is fine. For submission, use a persistent public host such as Render, Fly.io, Railway, a VPS, or another HTTPS-capable platform.

The production command should run the HTTP MCP server:

```bash
npm run build
PUBLIC_BASE_URL=https://your-real-domain.example npm run serve:prod
```

Required public endpoints from this project:

- `GET /healthz`
- `POST /mcp`
- `GET /.well-known/mcp-app.json`
- `GET /connector-card.json`
- static widget assets from `dist/`

## 2. Use a stable domain, not a temporary tunnel

Submission should use a URL that will not change between reviews or after approval. Prefer a custom domain or stable hosting subdomain.

Set one of these environment variables on the host:

```bash
PUBLIC_BASE_URL=https://your-domain.example
# or
PUBLIC_ORIGIN=https://your-domain.example

# Required for app submission widget templates.
WIDGET_DOMAIN=https://app.betterquizzes.com
```

Then verify that the server logs show the public MCP endpoint correctly. For submission, `WIDGET_DOMAIN` must be a stable HTTPS origin that you control. Do not use a temporary tunnel for this value.

## 3. Rebuild before every hosted release

After source changes:

```bash
npm ci --no-audit --no-fund
npm run build
npm run serve:prod
```

For hosted deployment, the build must happen either locally before upload or in the provider build command.

## 4. Run the project validation scripts

Before submission, run:

```bash
npm run build
npm run typecheck
node scripts/test-shared.mjs
node scripts/contract-test.mjs
node scripts/v1-regression.mjs
node scripts/host-readiness.mjs
node scripts/host-trial-doctor.mjs
node scripts/sdk-alignment-test.mjs
```

For public host validation, run after setting `PUBLIC_BASE_URL`:

```bash
set PUBLIC_BASE_URL=https://your-domain.example
npm run host:public:strict
npm run trial:public
```

## 5. Confirm widget/app metadata

Check that the MCP tool metadata includes:

- `openai/outputTemplate`
- `openai/widgetAccessible`
- a current `ui://widget/...` resource URI
- `openai/widgetDomain`
- `_meta.ui.domain`
- `text/html;profile=mcp-app` resource MIME type
- an accurate widget description
- current version/build strings

The `add_question` tool should remain storage-only. For assistant-authored quizzes, the model should call `open_quiz` once immediately after the first accepted `add_question`; later accepted questions should update the already-launched widget through token-scoped stored-draft polling. Keep `open_quiz` as the idempotent render tool for staged drafts and stored quizzes, and keep `create_quiz` described as a compatibility opener for complete user-supplied quiz packets.

## 6. Prepare user-facing submission materials

Likely required or strongly recommended:

- app name: BetterQuizzes
- short description
- longer description
- icon/logo
- screenshots or demo video/GIF
- support/contact email
- privacy policy URL
- terms of service URL, if required
- explanation of data handling
- age/safety positioning if the app may be used by students

## 7. Privacy and data handling

Document clearly:

- quiz content is generated by the model/user request
- user answers are sent back to ChatGPT for grading
- whether your server stores quizzes/submissions
- how long data is retained
- how logs are handled
- whether any third-party analytics are used

For the current architecture, be especially clear that the widget collects answers, confidence, and timing, then returns a structured submission capsule for grading.

## 8. Security hardening before review

Recommended production hardening:

- HTTPS only
- no dev-only console secrets
- no hardcoded private URLs
- rate limiting if hosted publicly
- request size limits
- safe error messages
- stable CORS/security headers
- no broad filesystem access
- no unnecessary user data storage

## 9. Manual review checklist

Before submitting, test:

- fresh quiz creation
- all supported question renderers
- interrupted loading recovery
- submit to ChatGPT
- reopening a submitted quiz
- one-question quiz layout
- skip flow
- malformed quiz diagnostics
- mobile/narrow layout
- public URL from a different browser/session

## 10. Submission-readiness rule of thumb

Do not submit while using ngrok as the final URL. Submit when BetterQuizzes is on a stable HTTPS host, the public strict tests pass, privacy/support materials exist, and a fresh ChatGPT chat can launch and submit a quiz without local-only dependencies.
